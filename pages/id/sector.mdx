# Sektor

Endpoint `GET /api/v1/sector` memungkinkan Anda untuk mengambil daftar sektor. Setiap sektor mencakup `name` dan `slug` yang dapat digunakan untuk memfilter data dalam permintaan API lainnya.

## Permintaan HTTP

### GET `/api/v1/sector`

Endpoint ini mengambil semua sektor yang tersedia.

### Contoh Permintaan

```bash
GET https://next-ksd.pekalongankab.go.id/api/v1/sector
```

### Parameter Query

Endpoint ini tidak menerima parameter query.

### Contoh Respons

```json
[
  {
    "_id": "66e290869729bf5d8017fff1",
    "name": "Ekonomi",
    "slug": "ekonomi"
  },
  {
    "_id": "66e290869729bf5d8017fff2",
    "name": "Energi",
    "slug": "energi"
  },
  {
    "_id": "66e290869729bf5d8017fff3",
    "name": "Geografis",
    "slug": "geografis"
  },
    ...
  {
    "_id": "66e290869729bf5d8017fffe",
    "name": "Tata Kelola Pemerintahan",
    "slug": "tata-kelola-pemerintahan"
  },
]
```

### Kode Respons

- **200 OK**: Permintaan berhasil dan mengembalikan daftar sektor.
- **400 Bad Request**: Terjadi kesalahan pada server saat memproses permintaan.

## Penggunaan

Anda dapat menggunakan kolom `slug` dari sektor yang dikembalikan oleh endpoint ini untuk memfilter atau merujuk sektor tertentu dalam kueri API lainnya (misalnya, memfilter elemen data berdasarkan sektor).

### Catatan

- Kolom `name` adalah nama sektor yang ramah pengguna.
- Kolom `slug` adalah pengidentifikasi aman URL yang digunakan untuk memfilter dan kueri.

## Penanganan Kesalahan

Jika terjadi kesalahan server atau permintaan gagal, respons akan mengembalikan kode status 400 dengan pesan yang menunjukkan kegagalan.

### Contoh Respons Kesalahan

```json
{
  "error": "Gagal mengambil data."
}
```
